FULL TECHNICAL STACK
---------------------------------------------------------------------
FRONTEND
  Framework:     React 18.3 + TypeScript
  Build tool:    Vite 5.4
  Styling:       Tailwind CSS 3.4, shadcn/ui (Radix UI)
  Charts:        Recharts 2.15
  PDF export:    jsPDF 4
  Mobile:        Capacitor 8 (iOS + Android)
  Routing:       React Router DOM 6
  State/data:    TanStack React Query 5
  Forms:         React Hook Form 7 + Zod

BACKEND (Self-hosted Supabase)
  Database:      PostgreSQL 15
  REST API:      PostgREST v12
  Auth:          GoTrue v2 (Supabase Auth)
  Realtime:      Supabase Realtime v2
  Storage:       Supabase Storage API v1
  Edge Runtime:  Supabase Edge Runtime v1 (Deno)
  API Gateway:   Kong 2.8
  Admin UI:      Supabase Studio

INFRASTRUCTURE
  OS:            Ubuntu 22.04 LTS
  Container:     Docker Compose
  Reverse proxy: Nginx (TLS, rate limiting, IP whitelist)
  Firewall:      UFW + Fail2ban
  Backups:       Daily pg_dump, 30-day retention, gzip
  Data center:   Lesotho Government — Mohale's Hoek

KEY SQL FUNCTIONS:
  update_updated_at_column()   — trigger for updated_at timestamps
  handle_new_user()            — auto-creates profile on auth signup
  get_user_role()              — returns current user's staff_role
  get_user_district_id()       — returns current user's district UUID
  get_user_center_id()         — returns current user's center UUID
  build_pedigree_cache(animal_id) — populates pedigree_records up to 4 generations
  calculate_genetic_indices(district_id, species, year, center_id)
    — computes annual_mortality_rate = deaths/avg_herd × 100
       annual_offtake_rate = (sales+slaughter+stolen+gifts_out
                              −gifts_in−purchases)/avg_herd × 100
       lambing_kidding_rate = parturitions/female_animal_days × 365
  recalculate_reproductive_indices(district, period_start, period_end, species)

EDGE FUNCTIONS (Deno):
  wahis-export            — POST, formats woah_disease_reports to WAHIS JSON
  process-offline-queue   — POST, applies pending offline_sync_queue entries
  calculate-genetic-indices — POST/scheduled, calls calculate_genetic_indices
                              across all districts

VIEWS:
  livestock_summary        — flat view joining districts, centers, farmers
  district_health_summary  — WOAH case counts per district

---------------------------------------------------------------------
SECURITY MODEL
---------------------------------------------------------------------
- Every table has RLS enabled
- Helper functions (SECURITY DEFINER) used in policies:
  get_user_role(), get_user_district_id(), get_user_center_id()
- Permission matrix:
  system_admin:   SELECT/INSERT/UPDATE/DELETE on all tables
  center_manager: Full access within their center
  district_officer: SELECT all in district, INSERT/UPDATE in district
  veterinarian:   SELECT all, INSERT/UPDATE health/vaccination/WOAH
  field_worker:   SELECT/INSERT in their district only
