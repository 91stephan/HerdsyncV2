SYSTEM CONTEXT — READ THIS CAREFULLY BEFORE WRITING
=====================================================================

PROJECT NAME: HerdSync V2 — Lesotho National Breeding System
CLIENT: Ministry of Agriculture and Food Security, Kingdom of Lesotho
TENDER REFERENCE: LSO-2000003942-0137-CS-QCBS
  "Digitalisation of National Breeding Centers" under the WaMCoP Project

PURPOSE: Replace manual paper-based livestock management at national
breeding centers with a digital system that manages Merino sheep and
Angora goat genetics across all 10 Lesotho districts.

---------------------------------------------------------------------
TECHNICAL STACK
---------------------------------------------------------------------
Frontend:    React 18 + TypeScript, Vite, Tailwind CSS, shadcn/ui,
             Capacitor (iOS/Android mobile), Recharts
Backend:     Supabase (PostgreSQL 15, PostgREST, GoTrue Auth,
             Realtime, Storage, Deno Edge Functions)
Deployment:  Self-hosted Docker Compose on Lesotho Government servers
             (Mohale's Hoek Data Center)
Auth:        JWT-based, Row Level Security (RLS) on all tables

---------------------------------------------------------------------
DATABASE SCHEMA (KEY TABLES)
---------------------------------------------------------------------
districts               — 10 Lesotho administrative districts
breeding_centers        — National breeding centers per district
profiles                — Ministry staff (system_admin, center_manager,
                          district_officer, field_worker, veterinarian)
farmers                 — Rural smallholder participants
livestock               — Animals with 15-digit ISO 11784 national RFID
                          ID, fiber metrics (greasy fleece weight, micron
                          diameter, staple length, clean yield %), pedigree
                          linkage (sire_id, dam_id), inbreeding coefficient
pedigree_records        — Materialised multi-gen ancestor cache
breeding_records        — Mating events
birthing_records        — Birth outcomes
reproductive_indices    — Period-based lambing/kidding rates
genetic_indices         — Annual Basotho indices: mortality rate,
                          offtake rate, lambing/kidding rate per animal-day
livestock_movements     — Tracks sales, slaughter, stolen, gifts, purchases
culling_exchange_records — Farmer low-yielding animal collection +
                           replacement distribution workflow
health_records          — Veterinary events
vaccination_records     — Vaccination schedule
woah_disease_reports    — WOAH/OIE disease notifications
compliance_documents    — Ministry document vault
tasks                   — Staff task assignments
offline_sync_queue      — Field-to-server sync for remote areas
liamis_sync_log         — LIAMIS integration audit
project_mis_sync_log    — WaMCoP MIS integration audit
efi_export_records      — Ethical Fashion Initiative fiber data exports
notifications           — In-app staff notifications
audit_log               — Immutable change audit trail

---------------------------------------------------------------------
EDGE FUNCTIONS (DENO / SERVER-SIDE)
---------------------------------------------------------------------
wahis-export            — Formats disease reports to WAHIS/OIE JSON
process-offline-queue   — Applies offline field records on reconnect
calculate-genetic-indices — Recalculates annual Basotho indices

---------------------------------------------------------------------
INTEGRATION POINTS
---------------------------------------------------------------------
- LIAMIS (Lesotho Integrated Agriculture MIS) — outbound data push
- WaMCoP Project MIS — scheduled sync
- WAHIS/OIE — disease event export (24h reporting requirement)
- EFI (Ethical Fashion Initiative) — fiber data for Basotho Brand

---------------------------------------------------------------------
SECURITY
---------------------------------------------------------------------
- Row Level Security on every table
- Role hierarchy: system_admin > center_manager > district_officer
  > veterinarian > field_worker
- JWT auth with refresh tokens
- Nginx TLS reverse proxy, UFW firewall, Fail2ban, no root SSH
- Immutable audit_log table
